# 🍨 سیستم سفارش آنلاین کافه‌بستنی

مشتری با اسکن QR بیرون کافه منو را باز می‌کند، سفارش می‌دهد و یک QR مخصوص سفارشش می‌گیرد. داخل کافه، صندوق‌دار آن QR را با وب‌کم لپ‌تاپ اسکن می‌کند، سفارش کامل را می‌بیند و تحویل را ثبت می‌کند. منو هم از پنل مدیریت قابل ویرایش است و تغییرات فوراً روی منوی مشتری اعمال می‌شود.

مشخصات کامل در [`spec.md`](spec.md) و ریز تسک‌ها در پوشه [`plan/`](plan/) است.

## پیش‌نیازها

- Python 3.10 یا جدیدتر
- Node.js 18 یا جدیدتر

## ۱) راه‌اندازی بک‌اند (FastAPI)

در ترمینال اول (از ریشه پروژه):

```bat
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
uvicorn app.main:app --reload
```

- سرور روی `http://127.0.0.1:8000` بالا می‌آید و Swagger UI در `http://127.0.0.1:8000/docs` است.
- بار اول، دیتابیس `cafe.db` خودکار ساخته و با منوی نمونه پر می‌شود.
- ⚠️ داخل فایل `.env` مقدار `PANEL_PASSWORD` (رمز پنل‌ها، پیش‌فرض `cafe1234`) و `SECRET_KEY` را حتماً عوض کنید.

## ۲) راه‌اندازی فرانت (React)

در ترمینال دوم:

```bat
cd frontend
npm install
npm run dev
```

فرانت روی `http://localhost:5173` بالا می‌آید و درخواست‌های `/api` را خودش به بک‌اند پراکسی می‌کند (نیازی به تنظیم اضافه نیست).

## صفحه‌ها

| آدرس | صفحه |
|---|---|
| `/` | منوی مشتری (مقصد QR بیرون کافه) |
| `/order/<کد>` | بلیت سفارش مشتری + QR |
| `/login` | ورود به پنل‌ها |
| `/cashier` | پنل صندوق: اسکن QR، سفارش دستی، سفارش‌های امروز |
| `/admin` | پنل مدیریت منو: دسته‌ها، محصولات، قیمت‌ها |

## اتصال موبایل مشتری‌ها (شبکه local)

1. IP لپ‌تاپ را با دستور `ipconfig` پیدا کنید (مثلاً `192.168.1.20`).
2. موبایل و لپ‌تاپ باید روی یک وای‌فای باشند.
3. آدرس منوی مشتری: `http://192.168.1.20:5173`
4. با همین آدرس یک QR کد بسازید (با هر ابزار ساخت QR)، چاپ کنید و بیرون کافه بگذارید.
5. اگر موبایل وصل نشد، در فایروال ویندوز پورت `5173` را برای شبکه خصوصی باز کنید (بار اول اجرای Node معمولاً ویندوز خودش می‌پرسد — «Allow» بزنید).

## نکته مهم وب‌کم (پنل صندوق)

مرورگر فقط روی **localhost یا HTTPS** اجازه دوربین می‌دهد. پس پنل صندوق را روی خود لپ‌تاپ با آدرس `http://localhost:5173/cashier` باز کنید تا اسکنر QR کار کند. اگر اسکن ممکن نبود، کد ۸ رقمی زیر QR مشتری را دستی وارد کنید.

## چرخه وضعیت سفارش

```
pending (ثبت آنلاین) → preparing (بعد از اسکن) → ready (آماده شد) → completed (تحویل و تسویه)
و در صورت نیاز: cancelled (لغو از pending یا preparing)
```

## بکاپ

کل دیتابیس یک فایل است: `backend/cafe.db` — کپی گرفتن از همین فایل یعنی بکاپ کامل سفارش‌ها و منو.
