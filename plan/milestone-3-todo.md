# Milestone 3 — QR کد سفارش + پنل صندوق

هدف: مشتری بعد از ثبت سفارش QR واقعی ببیند؛ صندوق‌دار با لپ‌تاپ لاگین کند، QR را با وب‌کم اسکن کند، سفارش را ببیند و وضعیتش را مدیریت کند؛ و سفارش حضوری (walk-in) را دستی ثبت کند.

## نمایش QR به مشتری
- [x] تکمیل OrderSuccessPage: نمایش تصویر QR (base64 از API) + کد متنی زیر آن برای ورود دستی
- [x] دکمه «مشاهده مجدد سفارش» — صفحه با آدرس `/order/:code` همیشه قابل بازکردن باشد (اگر مشتری صفحه را بست)

## لاگین پنل‌ها
- [x] LoginPage: فرم رمز عبور → `POST /api/auth/login` → نگهداری توکن در state/sessionStorage
- [x] گارد مسیرهای `/cashier` و `/admin`: بدون توکن → هدایت به لاگین
- [x] افزودن هدر Authorization به درخواست‌های محافظت‌شده در `api/client.ts`
- [x] هندل انقضا/نامعتبر بودن توکن (401 → بازگشت به لاگین)

## پنل صندوق (CashierPage) — تب اسکن
- [x] نصب `html5-qrcode` و کامپوننت اسکنر با وب‌کم لپ‌تاپ
- [x] بعد از اسکن موفق: `GET /api/cashier/orders/{code}` → نمایش کامل سفارش (اقلام، تعداد، قیمت واحد، جمع کل، اسم مشتری، توضیحات، وضعیت)
- [x] گذار خودکار pending→preparing سمت سرور + نمایش وضعیت جدید
- [x] ورودی متنی جایگزین: تایپ دستی کد سفارش (اگر QR اسکن نشد)
- [x] هندل کد نامعتبر/تکراری (سفارشی که قبلاً completed شده) با پیام واضح
- [x] دکمه‌های تغییر وضعیت: «آماده شد» (ready)، «تحویل و تسویه شد» (completed)، «لغو» (cancelled)

## پنل صندوق — تب سفارش دستی (walk-in)
- [x] فرم انتخاب از منو (همان دیتای `GET /api/menu`) با +/− تعداد و جمع کل زنده
- [x] فیلد اختیاری اسم مشتری و توضیحات
- [x] ثبت با `POST /api/cashier/orders` (وضعیت preparing) و نمایش رسید روی صفحه

## پنل صندوق — تب سفارش‌های امروز
- [x] لیست سفارش‌های امروز با فیلتر وضعیت (`GET /api/cashier/orders`) + رفرش خودکار هر ۱۵ ثانیه
- [x] کلیک روی هر سفارش → جزئیات + دکمه‌های تغییر وضعیت
- [x] نشان رنگی وضعیت‌ها (pending زرد، preparing آبی، ready سبز، completed خاکستری، cancelled قرمز)

## تست و تحویل
- [x] تست end-to-end واقعی: ثبت سفارش با موبایل → اسکن QR موبایل با وب‌کم لپ‌تاپ → دیدن سفارش → بردن تا completed
- [x] تست سفارش دستی walk-in و دیدنش در لیست امروز
- [x] نکته فنی: دسترسی وب‌کم در مرورگر نیاز به HTTPS یا localhost دارد — مستند کردن راه‌حل (اجرای پنل روی localhost لپ‌تاپ)

✅ معیار پذیرش: کارفرما یک سفارش از موبایل ثبت کند، QR آن را با وب‌کم لپ‌تاپ اسکن کند، سفارش کامل را روی لپ‌تاپ ببیند و تحویل را ثبت کند؛ همچنین یک سفارش حضوری دستی ثبت کند.
