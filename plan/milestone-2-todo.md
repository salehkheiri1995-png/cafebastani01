# Milestone 2 — صفحه منوی مشتری (React)

هدف: صفحه منوی موبایل‌فرست RTL که مشتری با اسکن QR بیرون کافه باز می‌کند، آیتم انتخاب می‌کند و سفارش ثبت می‌کند. (نمایش QR سفارش به‌صورت کامل در Milestone 3 تکمیل می‌شود؛ اینجا صفحه موفقیت ساده با کد سفارش کافی است.)

## راه‌اندازی فرانت
- [x] ساخت پروژه `frontend/` با Vite + React + TypeScript
- [x] نصب و تنظیم Tailwind CSS با پشتیبانی RTL (`dir="rtl"` روی html)
- [x] فونت فارسی Vazirmatn (لوکال یا CDN)
- [x] React Router: مسیرهای `/` (منو) و `/order/:code` (موفقیت سفارش)
- [x] `api/client.ts`: لایه fetch با base URL از env + هندل خطا
- [x] `types.ts`: تایپ‌های Category، Product، Order، OrderItem هم‌راستا با اسکیماهای بک‌اند

## صفحه منو (MenuPage)
- [x] فراخوانی `GET /api/menu` موقع بارگذاری + refetch هر ۳۰ ثانیه (اعمال فوری تغییرات مدیر)
- [x] نمایش دسته‌ها به‌صورت تب یا اسکرول افقی با ترتیب display_order
- [x] کارت محصول: نام، توضیح، قیمت با جداکننده هزارگان و «تومان»
- [x] دکمه +/− برای تعداد هر آیتم (state سبد در React — بدون localStorage)
- [x] نوار سبد چسبان پایین صفحه: تعداد اقلام + جمع کل + دکمه «ثبت سفارش»
- [x] حالت‌های loading و خطای شبکه با پیام فارسی

## ثبت سفارش
- [x] مودال/صفحه مرور سفارش: لیست اقلام، جمع کل، فیلد اختیاری اسم مشتری و توضیحات
- [x] `POST /api/orders` و هدایت به `/order/:code`
- [x] صفحه موفقیت (OrderSuccessPage): پیام «سفارش ثبت شد»، کد سفارش درشت، اقلام و جمع کل + یادآوری «این صفحه را داخل کافه نشان دهید» (تصویر QR در M3 اضافه/تکمیل می‌شود)
- [x] هندل خطای ثبت (مثلاً محصولی که همین الان ناموجود شد) با پیام واضح
